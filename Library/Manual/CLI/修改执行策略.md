# 问题说明

Windows 系统默认设置禁止在系统上运行脚本。

可通过 PowerShell 修改执行策略将脚本放行。

# 临时绕过执行策略

临时绕过执行策略的写法通常是这样：


```powershell
powershell -ExecutionPolicy Bypass -NoProfile -Command "codex"
````

如果你是在当前 PowerShell 窗口里直接执行，也可以先启动一个 **Bypass** 会话：

```powershell
powershell -ExecutionPolicy Bypass
```

然后在这个窗口里再运行：

```powershell
codex
```

更常见、也更稳一点的方式是直接对当前进程放行再跑命令：

```powershell
Set-ExecutionPolicy Bypass -Scope Process
codex
```

## 说明

- `-Scope Process` 只对当前窗口有效，关掉窗口就恢复原状。
    
- 这比改全局策略更安全，适合临时用一下。
    
- 如果 `codex` 仍然调用的是 `codex.ps1`，这个方式一般就能绕过报错。

# 直接修改执行策略

最常见的解决办法是把当前用户的执行策略改成 `RemoteSigned`，这样本地脚本可以运行：

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

如果只是想临时放行，也可以当前窗口用 `Bypass` 方式启动 PowerShell，
但日常开发更推荐 `RemoteSigned`。

## 你可以这样做

1. 以管理员身份打开 PowerShell，或者至少用当前用户范围修改策略。
    
2. 执行以下命令：
	
```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```
    
4. 输入 `Y` 确认后，再运行脚本。
    

## 如果还不行

- 先检查当前策略：`Get-ExecutionPolicy`。
	
- 如果公司电脑或组策略锁死了执行策略，可能需要管理员权限才能改。
