推荐用 `uv` 管理 Python 依赖。你在 Windows 上可以这样装。

**方式一：PowerShell 安装，推荐**

注意：不要以直接双击 PowerShell 的方式打开，

而是：手动按 `Win + R`，输入`powershell`后敲回车，然后执行官方推荐命令：

```powershell
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```

这个写法来自 uv 官方安装文档，比直接执行下面这个更稳：

```powershell
irm https://astral.sh/uv/install.ps1 | iex
```

安装完成后，系统提示设置环境变量，但不一定要手动执行，取决于你接下来怎么用终端。

推荐方式：关闭终端，重新打开。

如果你不想重开终端，

当前在 PowerShell，就执行：

```
$env:Path = "C:\Users\Administrator\.local\bin;$env:Path"
```

当前在 cmd，就执行：

```
set Path=C:\Users\Administrator\.local\bin;%Path%
```

当前在 Git Bash 里，就执行：

```
export PATH="/c/Users/Administrator/.local/bin:$PATH"
```

目的只是为了让 PATH 生效，最省事的做法还是重开终端。

如果你想重开终端，

关闭当前 Git Bash，重新打开一个新的 Git Bash，然后验证：

```bash
uv --version
```

能看到版本号就表示安装成功了。

**方式二：如果你已经装了 Scoop**

```
scoop install uv
```

然后重新打开终端：

```
uv --version
```

**方式三：如果你已经装了 Python/pip**

```
pip install uv
```

然后：

```
uv --version
```

不过 Windows 上我更推荐第一种，最省心。

另外，如果你装完后 PowerShell 能识别 `uv`，但 Git Bash 还是提示找不到，通常是 PATH 没刷新。关闭 Git Bash 重新打开即可；不行就重启一下终端或电脑。
