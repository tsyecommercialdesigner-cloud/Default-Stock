要对技能、提示词和模型上下文协议三者进行对比，首先要清楚它们是什么。
下面我们先对三者的概念进行介绍，再对三者进行比较。

# 提示词（Prompt）

提示词是用户向大语言模型输入的、用于引导其生成预期响应的声明式指令或结构化信息，通过自然语言或者结构化文本精确描述任务目标、上下文背景、角色身份、约束条件及输出格式等，引导大语言模型生成符合预期的响应结果。

通俗来讲，提示词就是你与 AI 沟通时对它说的话，以及你希望它理解并完成任务的方式。例如，“国庆我要去北京旅游三天，我喜欢户外游，我的预算大概 2000 元，帮我做个规划”。

在任意大语言模型对话页面里的输入都是提示词。

---
# 技能（Agent Skills）

技能是一种用于为 Agent 扩展专门能力的开放标准，它通过将特定领域的知识、工具能力与任务执行流程进行模块化封装，形成可复用、可组合的能力单元，使 Agent 能够以可控、可复用的方式调用这些能力来完成明确边界内的任务，从而为 Agent 提供稳定的任务执行能力。

通俗来讲，技能就是 Agent 的“职业技能证书+工具箱+行业经验手册”。技能让 AI 从“会聊天的百科全书”真正进化为“能执行任务的数字员工”，也让非技术人员能像搭积木一样，组装出能够解决具体问题的 AI 应用。

借助技能，Agent 可按需加载、动态组合多元能力，并在规划与推理机制驱动下完成特定任务乃至复杂工作流，从而显著降低开发与使用门槛，全面提升问题解决效能。

---
# 模型上下文协议（Model Context Protocol）

模型上下文协议是一套标准化的大语言模型与外部工具、数据源及服务交互的通信规范，旨在为模型与外部能力建立统一、安全、可扩展的连接机制。通过定义统一的资源访问方式和工具调用规范，模型可以用一致的方式接入本地文件、数据库、API 等外部资源，从而提升系统的可扩展性与开发效率。

通俗来讲，模型上下文协议就是一套让 Agent 能够规范、顺畅地连接和使用外部工具的“统一标准”和“接口规则”。它可以类比为房间里的“标准插座”，不管是手机、计算机，还是吹风机、冰箱等电器，只要接口统一就可以直接插入使用。同样，AI 也可以通过同一套规范调用不同工具，无须为每个工具都单独设计连接方式。

---
# 三者的类比

三者的核心差异对比如下表所示。

<p align=center>提示词、技能与模型上下文协议的比较</p>

| 维度 | 提示词 | 技能 | 模型上下文协议 |
| :--- | :--- | :--- | :--- |
| **本质** | 用户意图的表达方式 | 模块化封装的能力单元 | 标准化通信与连接协议，提供统一接入规则 |
| **生命周期** | 短暂、一次对话有效 | 持久，可反复使用 | 长期，是系统基础设施 |
| **复用性** | 低，每次重写 | 高，可重复使用 | 极高，行业通用 |
| **目标** | 表达意图 | 封装能力，可靠执行 | 实现互通、动态扩展 |
| **类比** | “我要去机场”（表达意图） | “司机+导航+订票平台”（执行者，帮你完成任务） | “交通规则、车载系统接口”（连接规则，保证顺畅协作） |

从形象化的角度类比三者就是：

- 提示词是“口头指令”；
- 技能是“工具装备”；
- 模型上下文协议是“插座标准”。

从解决问题的过程环节来看就是：

- 提示词解决“说什么”；
- 技能解决“怎么做”；
- 模型上下文协议解决“怎么连”。

三者相互协同，共同构成 Agent 系统的完整能力闭环，推动其从问答工具跃升为自主问题解决者。
