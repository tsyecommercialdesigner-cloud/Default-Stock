*根据结构化思想设计提示词是一种卓有成效的提示工程策略。它使用明确的结构引导、内容引导和提示词编排设计来提升提示的可读性，帮助大语言模型更准确地理解任务，并生成稳定的、可复现的、符合预期的内容。这种设计在一定程度上降低了AI原生应用在大语言模型版本升级时和跨大语言模型厂商迁移应用的成本。*

---
# 概述

利用特定的格式和语法来设计提示词，主要采用的方式包括层次结构、输入和输出位置、有序列表和无序列表、解释或补充说明、不可分割、强调内容、语义槽位、内容边界、输出格式、字段名称和数据类型，以及输出长度控制。

通过结构引导设计，我们可以在不增加模型参数的情况下，显著提高大语言模型的输出质量。此外，结构引导设计还有助于调试和解释大语言模型的输出，从而帮助我们更好地理解大语言模型的行为和输出。

---
# 层次结构

层次结构是组织和呈现提示文本的重要方式。可以借鉴Markdown的语法规则，用#来明确区分不同层级的标题，以提升提升内容的层次清晰度，同时确保其与多数代码编辑环境无缝兼容。

对于系统提示词：

```markdown
# 角色
你是一名XX，专精XX领域，你擅长XX，对于XX有着深入的研究和理解。
# Skills
用自然语言来描述Skill，或JSON格式的Skill直接复制粘贴在这里。
# 工作方式
详细描述这个助手完成任务所采取的方式，可以具体到每一个步骤。
# 任务限制
将这个助手执行任务时所需要避免的项列举到这里，比如执行完成后进行内容审核，以及要屏蔽的敏感词等。注意这里的任务限制是全局性的，与用户提示词中所声明的任务限制有区别。
```


对于用户提示词：

```markdown
# 任务描述
详细描述任务的目标，为AI的行动指明方向。
# 背景知识
提供完成任务所需的背景知识。
# 任务限制
注意这里的任务限制是局部的，与系统提示词中所声明的任务限制有区别。
# 输入内容
这里输入你要求AI进行处理的源文本。
# 输出示例
如果存在对生成内容的范式要求，请具体写出来。
# 输出
->
```

# 常用的Markdown标记符号


以下是常用于提示词的结构引导设计的Markdown标记符号：

| 符号                   | 名称      | 用途                    |
| -------------------- | ------- | --------------------- |
| # 标题                 | 标题（分级）  | 用来划分提示词的结构和内容         |
| ->                   | 生成符号    | 用来标识需要大语言模型进行文本补全的区域  |
| - 列表项                | 无序列表    | 用来呈现并列关系的无序列表         |
| 1. 列表项               | 有序列表    | 用来呈现具有先后顺序或逻辑关系的列表    |
| `*`斜体`*`             | 斜体      | 用来表示来自引用或摘录的内容        |
| `**`强调`**`           | 粗体      | 用来标注需要强调的关键词组或句子      |
| 他的名字叫<姓名>            | 占位符     | 用来标注大语言模型在指定位置生成恰当的内容 |
| 他的名字叫__              | 占位符     | 用来标注大语言模型在指定位置生成恰当的内容 |
| `<START>`、`<END>`    | 内容边界    | 用来标记内容的边界，引导大语言模型生成内容 |
| (解释或补充信息)            | 解释/补充   | 用来对括号前的内容进行解释或补充说明    |
| {}                   | 生成内容占位符 | 用来标注需要填充的内容           |
| {%format}            | 格式化字符串  | 用来定义字段的数据类型           |
| (parameter : format) | 格式化字符串  | 用来定义字段的数据类型           |
| “文本内容”               | 不可分割    | 用来表示词组成名称不可分割         |
| "文本内容"               | 字符串引用   | 用来标注此文本内容为要进行原样输出的字符串 |

# 常用的文件输出格式


以下是常用的可以指定大语言模型为我们输出的文件格式：

| 格式名称            | 格式类型     | 用途                             |
| --------------- | -------- | ------------------------------ |
| Markdown        | 文本格式     | 底层内容的记忆、存储                     |
| HTML            | 编程语言（前端） | 内容可视化                          |
| CSS             | 级联样式表    | 指定网页外观的显示方式                    |
| CSV             | 结构化数据    | 适合将数据导入 Excel 或数据库             |
| XML             | 可拓展标记语言  | 适合传统企业级系统或特定配置文件               |
| YAML            | 可拓展标记语言  | 常用于配置文件，比 JSON 更利于人类阅读         |
| KV              | 结构化数据    | 表示关键词：值的最佳方式                   |
| JSON            | 结构化数据    | 创建Skill、表示数据结构等                |
| LaTeX           | 公式       | 用来输出复杂数学公式、物理方程式或学术论文排版        |
| Mermaid         | 图表       | 用来生成流程图、思维导图、时序图、甘特图等          |
| PlantUML        | 图表       | 类似于 Mermaid，用于生成 UML 图         |
| Graphviz        | 图表       | 用于生成复杂的关系图和网络拓扑图               |
| SVG             | 矢量图代码    | 用来绘制简单的图标或几何图形                 |
| SQL             | 结构化查询语言  | 用来在数据库进行数据查询                   |
| Regex           | 正则表达式    | 用来生成并解释特定模式的正则匹配代码             |
| Shell/Bash      | CLI命令    | 用于服务器运维和自动化命令                  |
| Python/Java     | 编程语言（后端） | 用于自动化办公、网络爬虫等                  |
| JavaScript      | 全栈式语言    | 开发网页、服务器、手机 App、桌面软件甚至是控制硬件    |
| Outline         | 大纲/思维导图  | 按层级结构（I, II, III, a, b, c）列出提纲 |
| Step-by-step    | 分步指南     | 用来要求大语言模型按顺序明确写出每一个步骤          |
| Q&A / FAQ       | 问答对      | 用来将长文章总结成“问题-答案”的格式            |
| Script/Dialogue | 剧本/对话    | 用来指定大语言模型输出按角色名加冒号格式的对话        |
| Email           | 电子邮件     | 用来指定生成包含主题、称呼、正文、落款的电子邮件       |
| Hashtags        | 表情符号和标签  | 用来指定生成推文/社交媒体帖子等带 #标签 的内容      |
