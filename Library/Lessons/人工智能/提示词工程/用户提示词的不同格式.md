# 自然语言格式

*自然语言是指日常生活中人与人之间对话采用的语言。可以通过自然语言格式的提示词与LLM进行沟通。*

自然语言具有以下特点：
- 非结构化：自然语言表达方式极为灵活。例如，用户能够根据自身需求随意提问，如”今天天气怎么样？“或者”帮我写一段关于AI的文章。“
- 依赖上下文：在通过自然语言与LLM进行沟通时，语境常常依赖具体的上下文。例如，“苹果”在不同的语境中，既可能指水果，也可能指公司名称。
- 存在歧义：同一句话可能会被理解为不同意思。例如，“他借了我五块钱。”这句话既可以被理解为“他把五块钱借给了我”，也可以被理解为“我把五块钱借给了他”。
- 多样性：对于同一个意思，可以采用不同的自然语言来表达。例如，”能帮我吗？“和”请协助我完成这个任务。“虽然内容不同，但表达的是相同的意思。

> [!info] 自然语言格式的提示词示例
>你是一名图像批处理小助手，你擅长批量修改图像的画面内容、文案、宽高比及图像大小等。
>我现在要更改一批SKU图片，需要制作包括以下数量规格的SKU：
>1卷66只
>2卷132只
>3卷198只
>4卷264只
>5卷330只
>6卷396只
>请根据以上所提供的数量规格，为我制作相应的SKU图片。
>最终生成的图片，请参考格式如下：
>图像宽高比 = 1：1
>图像像素尺寸 = 800px * 800px

# Markdown格式 

*Markdown格式是一种轻量级的文本格式。它借助简单的符号和语法，让用户能够以结构化的方式表达内容，使文本更具可读性、更加清晰。使用Markdown格式提示词与LLM进行沟通，便于AI理解复杂信息。*

Markdown格式具有以下特点。
- 以结构化的方式呈现内容：借助标题、列表、代码块等语法，可将信息分层分类，避免冗长的纯文本造成理解上的混乱。
- 强调重点：通过加粗、斜体、下划线等方式突出关键信息，有助于AI快速捕捉用户意图。
- 便于展示代码：采用代码格式能够清晰呈现代码，避免因格式错误引发误解。
- 跨平台兼容：Markdown格式属于纯文本格式，几乎所有文本编辑器和AI平台都支持Markdown格式，便于内容的迁移和共享。

> [!info] Markdown格式的提示词示例

> 你是一名图像批处理小助手，你擅长批量地修改图像的画面内容、文案、宽高比及图像大小等，你的任务是帮我更改一批SKU图片。

> - 1卷66只
> - 2卷132只
> - 3卷198只

> - 6卷396只
> # 任务要求
> 仅更改位于图片右下角的描述商品数量规格的部分。
> # 任务限制
> - 图像宽高比为1:1
> - 图像像素尺寸为800px*800px
> # 输出
> ->

# 结构化标识格式

*可以在提示词中加入一些结构化标识，以便内容更好被理解。
例如，使用【】或「」等符号来表示标题，以此定义角色、任务、场景、诉求等；利用”1、2、3“等来表示步骤内容；利用多个”-“符号来表示无序列表中的并列内容。*

> [!info] 结构化标识格式的提示词示例
> 【角色】
> 你是一名图像批处理小助手，擅长批量地修改图像的画面内容、文案、宽高比及图像大小等。
> 
> 【数量规格】
> - 1卷66只
> - 2卷132只
> - 3卷198只
> - 4卷264只
> - 5卷330只
> - 6卷396只
>   
> 【任务】
> 请根据所提供的数量规格，为我按顺序依次修改图片中位于底部的标识着数量规格的区域，然后逐张地生成全新的图片。
> 
> 【要求】
> - 图像的宽高比是1：1
> - 图像的像素尺寸是800px * 800px

# XML格式

*XML和YAML一样都是一种可扩展标识语言，用于存储和传输结构化数据。它借助自定义的标签（如`<tag>`)来定义数据的层级和语义，从而让信息以树状结构呈现。*

XML格式适用于复杂的数据和任务。XML格式能够让LLM精准解析复杂信息。

相较于其他格式的提示词，XML格式的提示词更为复杂，可读性也较差。
XML格式对具备编程能力的用户相对友好，没有编程能力的用户使用XML格式的提示词难度较大。

> [!info] XML格式的提示词示例
```
<role>你是一名图像批处理小助手，擅长批量地修改图像的画面内容、文案、宽高比及图像大小等。</role>
<background>我是一名电商运营，我要修改并制作一批SKU图片。</background>
<specifications>
 - 1卷66只
 - 2卷132只
 - 3卷198只
 - 4卷264只
 - 5卷330只
 - 6卷396只
</specifications>
<instruction>
请根据所提供的数量规格，为我按顺序依次修改图片中位于底部的标识着数量规格的区域，然后逐张地生成全新的图片。
</instruction>
<format>
Aspect Ratio = 1:1
Pixel Size = 800px*800px
<format>
```

# 形式语言风格提示

*在某些场景中，单纯使用自然语言编写提示并不足以准确描述复杂的逻辑结构和处理流程。形式语言的严谨性和精确性为提示工程提供了重要的启示。*

将编程风格的形式语言巧妙地融入提示，以期在AI原生应用开发中取得更为出色的效果。
我们用==flp(Formal Language for Prompt)==来表示形式语言风格的提示词。
使用形式语言风格的提示词的好处是可以精确定义不同的数据类型，从而有效避免数据类型错误和混淆，大幅提升提示的可读性和可维护性。

## 编译器提示

*为了使大语言模型准确识别并执行任务，需要提供明确的编译器提示。该提示告诉大语言模型我们正在使用哪种编程语言，以便其根据相应的语法和语义规则来解析和执行代码，编译器提示与Shell 和Python脚本文件开头的声明类似。*

```
#!/bin/bash 
#!/usr/bin/env python 
```

可以使用以下格式来给出编译器提示。

`````
#! flp(Formal Language for Prompt)是一种自适应提示的形式语言。这种编程语言供大语言模型生成和优化自适应提示，请你扮演flp解释器模拟输出，**除非程序异常，否则不要作出任何解释**
```flp 
这里是具体的代码。
```
# execute
<这里是代码调用或立即执行>
`````

有了编译器提示，我们就可以将自然语言以编程语言的形式组织起来。


````
#! flp(Formal Language for Prompt)是一种自适应提示的形式语言。这种编程语言供大语言模型生成和优化自适应提示，请你扮演flp解释器模拟输出，**除非程序异常，否则不要作出任何解释**
```flp 
// 这是指令，没有使用""标注，将它作为提示词解析并输出
content = 帮我写一篇关于垃圾袋的小红书种草风格的文案
// 这是字符串，使用""标注，将它原样输出
title = "绝绝子！姐妹们，今天给大家推荐一款用了都说好的垃圾袋————"
product_name = 居态艾草香氛抗菌免撕垃圾袋
```
# execute
print(title + product_name + content)
````

在这个例子中，用字符串变量来存储文案的标题、产品名称和正文，以便拼接和输出字符串。
